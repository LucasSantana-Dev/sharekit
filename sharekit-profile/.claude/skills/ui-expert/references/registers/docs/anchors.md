## When to use this register

Docs register applies when: the brief calls for developer documentation, API reference, technical guide, or knowledge base. The goal is clarity, scannability, and reference value. Register assumes an audience *looking up specific information*, not reading narrative prose.

## What this register forbids

- No sidebar nav absent or hidden on desktop (hamburger menu required = broken wayfinding).
- No code examples fewer than prose paragraphs (docs are code-primary; reverse the ratio).
- No decorative sidebars or off-topic CTAs in the main content column.
- No excessive explanatory copy obscuring the actual API/tool reference.
