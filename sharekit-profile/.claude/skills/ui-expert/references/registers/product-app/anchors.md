## When to use this register

Product-app register applies when: the brief calls for a post-login UI surface (workspace, dashboard, editor, settings panel). The goal is task completion and daily usability, not conversion or marketing. Register assumes a user who has already signed up and is *inside* the product doing work.

## What this register forbids

- No marketing-style copy ("Discover new features," "Why use X?") in core workflow.
- No upsell modals or "Upgrade" interruptions in task paths.
- No multiple competing CTAs for a single task (e.g., "Save," "Save & continue," "Save & review," "Cancel").
- No sidebar nav with 8+ items and 2–3 levels of drilldown.
- No decorative animations or motion that slows task execution.
