# Execution Flow Reference — pr-flow

Detailed command reference and sequencing for PR creation.

## Conventional Commit Format

Strictly enforced. Reject non-conforming messages in Step 1.

```
<type>(<scope>): <subject>
<blank line>
<body (optional)>
<blank line>
<footer (optional)>
```

- **type:** `feat`, `fix`, `docs`, `refactor`, `test`, `chore`, `perf`
- **scope:** optional, e.g., `(auth)`, `(api)`, `(ui)`
- **subject:** imperative, lowercase, no period, max ~50 chars
- **body:** wrapped at 72 chars, present tense, why/what not how
- **footer:** `Closes #123`, `Refs #456`, `Breaking-Change: ...`

**Example:**
```
fix(mcp): handle connection timeout in desktop worker

Previously, socket timeout would hang indefinitely. Now:
- Add 5s timeout to connect() call
- Emit explicit error on timeout
- Retry up to 3x before giving up

Fixes #789
```

## Branch Naming

Auto-generated from commit type + subject (Step 2).

**Slugification rules:**
- Lowercase all
- Replace spaces with `-`
- Strip punctuation (`:`, `.`, `,`, etc.)
- Max 50 chars total (truncate subject if needed)

**Examples:**

| Commit | Branch |
|--------|--------|
| `feat: add theme selector` | `feat/add-theme-selector` |
| `fix: desktop MCP connection` | `fix/desktop-mcp-connection` |
| `docs: Update CHANGELOG for v0.8.3` | `docs/update-changelog-v083` |
| `refactor(auth): clean up OAuth flow` | `refactor/auth-clean-up-oauth-flow` |

Override with `--branch <custom-name>`.

## Atomic Git Sequence (Step 3)

All-or-nothing: if any step fails, roll back and report.

```bash
set -e  # Exit immediately on error

# 1. Create + switch to branch
git checkout -b <branch> <base>

# 2. Stage (check for unintended files first)
git status --short  # <- Prompt user to verify
git add -A

# 3. Commit (verbatim message, NO trailer editing)
git commit -m "<message>"

# 4. Push with upstream tracking
git push -u origin <branch>
```

**Safety checks (pre-sequence):**
- Branch does not exist locally: `git rev-parse --verify <branch> 2>&1 | grep -q "fatal" || { echo "Branch exists"; exit 1; }`
- Branch does not exist remotely: `git ls-remote --heads origin <branch> | grep -q <branch> && { echo "Remote branch exists"; exit 1; }`

**Abort steps:**
- If `git commit` fails: do NOT `git commit --amend`; instead roll back (`git reset --soft HEAD~1`), surface error, halt.
- If `git push` fails: do NOT retry with `--force-push`; surface error, halt.

## PR Creation (Step 4)

Extract repo slug from git remote, call `gh pr create` with title + body.

### Repo slug extraction

```bash
ORIGIN=$(git remote get-url origin)

# GitHub https: https://github.com/owner/repo.git
# GitHub ssh: git@github.com:owner/repo.git
# GitHub short: github.com/owner/repo (no .git)

REPO_SLUG=$(echo "$ORIGIN" | sed -E 's|.*github\.com[:/]([^/]*)/([^/]*?)(?:\.git)?$|\1/\2|')

# Validate format (owner/repo)
[[ "$REPO_SLUG" =~ ^[^/]+/[^/]+$ ]] || { echo "BLOCKED: Could not parse repo slug from $ORIGIN"; exit 1; }
```

### Title generation

From commit message, extract type + subject; capitalize subject first word.

```bash
# Input: "fix: desktop MCP connection"
# Output: "fix: Desktop MCP connection"

COMMIT_MSG="<commit-message>"
TYPE=$(echo "$COMMIT_MSG" | sed -E 's/^([a-z]+)(\(.+\))?:.*/\1/')
SUBJECT=$(echo "$COMMIT_MSG" | sed -E 's/^[^:]*: (.*)/\1/')
SUBJECT_CAP="$(tr '[:lower:]' '[:upper:]' <<< ${SUBJECT:0:1})${SUBJECT:1}"
TITLE="$TYPE: $SUBJECT_CAP"
```

### PR body template

Minimal template (no AI attribution per CLAUDE.md):

```markdown
## Summary
<auto-generated from commit body or diff stat>

## Test plan
- [ ] CI passes
```

Optional: query knowledge-brain for prior templates (search `"PR body"` or `"PR summary"`).

### Full gh command

```bash
gh pr create \
  --repo "$REPO_SLUG" \
  --title "$TITLE" \
  --body "$(cat <<'EOF'
## Summary
<auto-generated summary of staged changes>

## Test plan
- [ ] CI passes
EOF
)" \
  $([ "$DRAFT" = "true" ] && echo "--draft" || true)
```

**Success:** gh returns PR URL + HTTP 201. Extract URL from output.

**Failure modes:**
- `gh auth` missing: direct to `gh auth login`
- Repo slug invalid: surface error, abort
- Base branch protection: surface gh error, abort

## Step 5: Report

On success, return:

```
PR created: <URL>
  Branch: <branch> → <base>
  Commits: 1
  Draft: <yes/no>
```

On failure, return first blocker + next action:

```
Blocked: <blocker>
  Action: <next step or how to recover>
  Example: git rebase <base> && git push -u origin <branch>
```
