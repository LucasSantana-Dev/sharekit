# sharekit backlog — 2026-06-23

Generated by `/backlog`. 12 findings (audit + build-context), ponytail-filtered (3 dropped, 2 merged). Fresh repo → all new.

## Phase 1 — correctness (cheap, ~2h total)

- [ ] **#1 Skip symlinks in `walk()`** — bug/security, high, S
  - Evidence: `src/sharekit.ts` `walk()` recurses into dirs; a symlink cycle in an untrusted fetched profile → infinite recursion/OOM; a symlink could also escape the target root. `copyFileSync` follows symlinks (copies target, not link).
  - Accept: `walk()` uses `lstatSync`, skips symlinks (don't follow, don't copy); add a test with a cyclic symlink that terminates.
- [ ] **#2 Preserve exec bit on copy** — bug, med, XS
  - Evidence: all copies use `fs.copyFileSync` which drops mode; a shipped `toggle.sh`/`setup.sh` loses `+x`.
  - Accept: `chmodSync(dest, statSync(src).mode)` after copy; test that an executable source stays executable.
- [ ] **#3 Friendly errors: git-missing + malformed TOML** — bug/robustness, med, XS
  - Evidence: `execSync("git …")` ENOENT is masked as "No profile at URL"; `TOML.parse` throws raw on a malformed `sharekit.toml`.
  - Accept: detect ENOENT → "git not found — install git"; wrap `TOML.parse` → "Invalid sharekit.toml: <msg>".
- [ ] **#4 `mkdir` parent before rollback restore** — bug, med, XS
  - Evidence: `rollback` `copyFileSync(src, a.dest)` assumes the parent dir exists; ENOENT if it was deleted between install and rollback.
  - Accept: `mkdirSync(dirname(dest), {recursive:true})` before restore copy; test rollback after parent dir removed.

## Phase 2 — features / coverage

- [ ] **#5 Integration test: install→backup→rollback + cursor/shared mirroring** — test, med, S
  - Evidence: only `plan()`/`init()` unit-tested; the install lifecycle and the `cursor/`+`shared/` roots have zero automated coverage.
  - Accept: a hermetic test that installs a temp profile (incl. cursor/ + shared/ files), asserts mirror destinations, then rolls back and asserts restore.
- [ ] **#6 `init` scaffolds `cursor/` and `shared/`** — feature, med, S
  - Evidence: `init()` only writes `claude/`; README promises cursorrules + dotfiles.
  - Accept: `init` accepts/optionally scaffolds `cursor/` + `shared/` skeletons; doc updated.
- [ ] **#9 `install <user>@<ref>` (branch/tag pinning)** — feature, med, S — **see ADR-0001**
  - Evidence: always HEAD today. `docs/decisions/0001-profile-versioning.md`: ship `git clone --depth 1 --branch <ref>` (one flag = tags + branches, shallow-native). Defer lockfile, reject semver.
  - Accept: `install <user>@<ref>` resolves a tag or branch; `install <user>` unchanged (HEAD); cache key includes ref; `@missing-ref` errors clearly; test both ref kinds.
- [ ] **#8 `--include-hooks` opt-in** — feature, low, S
  - Evidence: `settings.json` always skipped (security default); no reviewed opt-in even after preview.
  - Accept: `--include-hooks` installs `settings.json` after an explicit extra confirm; default stays skip.
- [ ] **#10 Backup pruning** — feature, low, S
  - Evidence: `~/.sharekit/backups/` grows unbounded.
  - Accept: auto-keep last N (default 5) per user on install, or a `prune` subcommand.

## Phase 3 — DX / CI

- [ ] **#11 Release automation (tag → CI publish)** — ci, med, M
  - Evidence: manual `npm publish` is painful (npm WebAuthn 2FA, non-TTY blocks). A tag-triggered publish via npm automation token / OIDC trusted publishing removes it.
  - Accept: pushing a `vX.Y.Z` tag publishes `@lucassantana/sharekit` from CI with no local 2FA step.
- [ ] **#12 CI: lint + node matrix** — ci, low, S
  - Evidence: CI runs build+test on Node 20 only; no lint.
  - Accept: matrix {20,22}; an eslint/format check step.
- [ ] **#7 Add `CHANGELOG.md`** — docs, low, XS
  - Accept: Keep-a-Changelog file seeded with 0.1.0.
