## Phase 1 ‚Äî correctness (cheap, ~2h total)

- [ ] **#1 Skip symlinks in `walk()`** ‚Äî bug/security, high, S
  - Evidence: `src/sharekit.ts` `walk()` recurses into dirs; a symlink cycle in an untrusted fetched profile ‚Üí infinite recursion/OOM; a symlink could also escape the target root. `copyFileSync` follows symlinks (copies target, not link).
  - Accept: `walk()` uses `lstatSync`, skips symlinks (don't follow, don't copy); add a test with a cyclic symlink that terminates.
- [ ] **#2 Preserve exec bit on copy** ‚Äî bug, med, XS
  - Evidence: all copies use `fs.copyFileSync` which drops mode; a shipped `toggle.sh`/`setup.sh` loses `+x`.
  - Accept: `chmodSync(dest, statSync(src).mode)` after copy; test that an executable source stays executable.
- [ ] **#3 Friendly errors: git-missing + malformed TOML** ‚Äî bug/robustness, med, XS
  - Evidence: `execSync("git ‚Ä¶")` ENOENT is masked as "No profile at URL"; `TOML.parse` throws raw on a malformed `sharekit.toml`.
  - Accept: detect ENOENT ‚Üí "git not found ‚Äî install git"; wrap `TOML.parse` ‚Üí "Invalid sharekit.toml: <msg>".
- [ ] **#4 `mkdir` parent before rollback restore** ‚Äî bug, med, XS
  - Evidence: `rollback` `copyFileSync(src, a.dest)` assumes the parent dir exists; ENOENT if it was deleted between install and rollback.
  - Accept: `mkdirSync(dirname(dest), {recursive:true})` before restore copy; test rollback after parent dir removed.
