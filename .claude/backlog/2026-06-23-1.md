# sharekit backlog — 2026-06-23 (run 2, post-0.2.0)

Second `/backlog` after 0.2.0 shipped. Sources: a fresh read-only audit of the 0.2.0 delta (3 findings) + a CodeRabbit CLI review of the full delta since `13c1ec0` (6 findings). Reconciled + deduped → 6 issues. Run-1 (`2026-06-23.md`, #1–12) all shipped/closed.

## Phase A — code (security + correctness)

- [ ] **Harden `fetchProfile` against shell injection** — security, high, S
  - Evidence: `src/sharekit.ts:60,73,75` interpolate `user`/`ref` into `execSync` shell strings (`sh -c`). An embedded `"` breaks out: `install 'x"; touch /tmp/pwned; "'` executes. Self-inflicted today (CLI arg), but sharekit fetches arbitrary user-named repos and may be scripted.
  - Accept: use `execFileSync('git', [...args, '--', url, dir])` (array args, no shell) for clone + pull; kills shell- and arg-injection. Adjust git-missing detection (`ENOENT` instead of `status 127`) and the cached-pull path. Source: CodeRabbit C1.
- [ ] **Fix flags-before-arg parsing + type opts + preview usage** — bug, med, XS
  - Evidence: `src/index.ts:38–50` reads `arg = rest[0]` before flag parsing, so `install --include-hooks user@ref` captures `--include-hooks` as the user. Also `opts: any` (line 18,45) violates the no-`any` standard; usage message (line 40) omits `[@<ref>]` for `preview`.
  - Accept: split flags from positionals (flags work in any position); replace `any` with `interface InstallOpts { includeHooks?: boolean }`; show `[@<ref>]` for both install and preview. Source: CodeRabbit C2/M2 + audit.

## Phase B — tests + docs

- [ ] **Real + expanded test coverage** — test, med, S
  - Evidence: `test/integration.test.ts:51–133` hand-rolls the apply/rollback flow instead of calling the exported `applyProfile`/`restoreBackup`, so it stays green even if those break — defeats the refactor's purpose. No tests for `pruneBackups` (keep-5) or the `--include-hooks` confirm flow.
  - Accept: integration test invokes the real `applyProfile`/`restoreBackup`; add `pruneBackups` test (7 backups → 5 remain, newest); add `--include-hooks` confirm test (accept + reject via mocked readline). Source: CodeRabbit M1 + audit.
- [ ] **README: document 0.2.0 features** — docs, med, XS
  - Evidence: README omits `install <user>@<ref>` pinning and `--include-hooks`; CLI help (`index.ts:9,15`) and CHANGELOG have them.
  - Accept: document both with examples in the Install section. Source: audit.
